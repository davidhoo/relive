# Relive 项目架构文档

## 整体架构

Relive 采用 **三层架构** 设计：

```
┌─────────────────────────────────────────┐
│          ESP32 墨水屏相框                │
│  (定时获取照片并显示)                    │
└───────────────┬─────────────────────────┘
                │ HTTP API
                ▼
┌─────────────────────────────────────────┐
│         Backend API 服务                 │
│  ┌─────────────────────────────────┐    │
│  │  REST API Layer (FastAPI)       │    │
│  └─────────────┬───────────────────┘    │
│                │                         │
│  ┌─────────────▼───────────────────┐    │
│  │  Service Layer                  │    │
│  │  - 照片扫描服务                  │    │
│  │  - AI 分析服务 (Qwen3-VL)       │    │
│  │  - 评分服务                      │    │
│  │  - 照片选择服务 (往年今日/最佳)  │    │
│  └─────────────┬───────────────────┘    │
│                │                         │
│  ┌─────────────▼───────────────────┐    │
│  │  Data Layer (SQLAlchemy ORM)   │    │
│  └─────────────┬───────────────────┘    │
└────────────────┼─────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│      Database (SQLite/PostgreSQL)       │
│  - photos 表                             │
│  - tags 表                               │
│  - display_history 表                    │
│  - settings 表                           │
└─────────────────────────────────────────┘
```

## 核心组件说明

### 1. **Backend 服务** (`/backend`)

#### 目录结构
```
backend/
├── src/
│   ├── api/              # API 路由层
│   │   ├── photos.py     # 照片相关接口
│   │   ├── display.py    # 展示相关接口（供ESP32调用）
│   │   └── admin.py      # 管理接口
│   ├── services/         # 业务逻辑层
│   │   ├── photo_scanner.py    # 照片扫描服务
│   │   ├── ai_analyzer.py      # AI 分析服务（调用 Qwen API）
│   │   ├── photo_selector.py   # 照片选择算法
│   │   └── scoring.py          # 评分逻辑
│   ├── models/           # 数据模型（ORM）
│   │   └── photo.py      # Photo, Tag, DisplayHistory 模型
│   ├── utils/            # 工具函数
│   │   ├── database.py   # 数据库连接和会话管理
│   │   ├── image.py      # 图片处理（EXIF 提取等）
│   │   └── logger.py     # 日志配置
│   └── config/           # 配置管理
│       └── settings.py   # 配置文件
├── main.py               # 应用入口
└── requirements.txt      # Python 依赖
```

#### 核心服务流程

**照片分析流程**:
```
1. PhotoScanner 扫描 NAS 目录
   ↓
2. 提取 EXIF 信息（拍摄时间、尺寸等）
   ↓
3. AIAnalyzer 调用 Qwen3-VL API
   ↓
4. 生成描述、文案、分类
   ↓
5. ScoringService 计算艺术性和回忆价值评分
   ↓
6. 保存到数据库
```

**照片选择流程** (供 ESP32 调用):
```
1. ESP32 请求 /api/display/photo
   ↓
2. PhotoSelector 根据策略选择照片:
   - 往年今日 (优先)
   - 最高评分
   - 随机高分照片
   ↓
3. 记录到 display_history
   ↓
4. 返回照片 URL 和元数据
```

### 2. **数据库设计** (`/database`)

#### 核心表结构

**photos** - 照片主表
- 存储照片文件信息
- AI 分析结果（描述、文案、分类）
- 评分（艺术性、回忆价值）
- 分析状态

**tags** - 标签表
- 照片标签（人物、地点、事件等）

**photo_tags** - 照片标签关联表
- 多对多关系

**display_history** - 展示历史
- 记录相框显示过的照片
- 展示原因（往年今日/最高分/随机）

**settings** - 配置表
- 系统配置（API Key、路径等）

## API 接口设计（待实现）

### 管理接口
- `POST /api/scan` - 触发照片扫描
- `GET /api/photos` - 查询照片列表
- `GET /api/photos/{id}` - 获取照片详情
- `POST /api/photos/{id}/analyze` - 重新分析照片

### 展示接口（供 ESP32 调用）
- `GET /api/display/photo` - 获取当前应显示的照片
  - 查询参数: `strategy=on_this_day|best|random`
- `GET /api/display/image/{id}` - 下载照片文件

## 技术选型

| 组件 | 技术 | 原因 |
|------|------|------|
| Web 框架 | FastAPI | 异步高性能，自动 API 文档 |
| ORM | SQLAlchemy | 功能强大，支持多数据库 |
| 数据库 | SQLite | 轻量级，易部署 |
| AI | Qwen3-VL API | 强大的视觉理解能力 |
| 图片处理 | Pillow | Python 标准图片库 |
| HTTP 客户端 | httpx | 异步支持 |
