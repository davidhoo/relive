### 1.2 架构特点

| 特点 | 说明 |
|------|------|
| **分层架构** | 清晰的用户层、应用层、存储层、外部服务层 |
| **微服务化** | 核心业务拆分为独立的 Service 模块 |
| **RESTful API** | 标准化的 HTTP 接口，支持多端接入 |
| **轻量级部署** | 单体应用，Docker 容器化，适合 NAS 环境 |
| **异步处理** | 照片扫描和 AI 分析异步执行 |
| **缓存优化** | Redis 缓存热点数据，提升性能 |
| **可扩展性** | 预留扩展接口，支持未来功能 |

## 三、核心模块设计

### 3.1 照片管理模块 (PhotoService)

**职责**：
- 照片元数据的增删改查
- 照片列表查询（分页、筛选、排序）
- 照片详情获取
- 照片评分更新
- 照片标签管理

**核心方法**：
```go
type PhotoService interface {
    // 列表查询
    ListPhotos(params QueryParams) (*PhotoListResponse, error)

    // 获取详情
    GetPhotoByID(id uint) (*Photo, error)

    // 更新照片
    UpdatePhoto(id uint, updates map[string]interface{}) error

    // 删除照片（软删除）
    DeletePhoto(id uint) error

    // 按条件查询
    FindPhotos(filters PhotoFilters) ([]*Photo, error)

    // 获取缩略图
    GetThumbnail(id uint, size int) ([]byte, error)
}
```

**设计要点**：
- ✅ 使用 Repository 模式隔离数据访问
- ✅ 缓存热点照片数据（Redis）
- ✅ 缩略图懒加载生成（首次访问时生成并缓存）
- ✅ 支持复杂查询条件组合

---

### 3.2 扫描调度模块 (ScanService)

**职责**：
- 全量扫描（首次）
- 增量扫描（日常）
- 断点续传
- 进度跟踪
- 速度控制

**扫描流程**：
```
开始扫描
  ↓
遍历目录（递归）
  ↓
发现新照片/修改照片
  ↓
提取 EXIF → 存入数据库（标记未分析）
  ↓
加入 AI 分析队列
  ↓
继续下一张
  ↓
完成/暂停/停止
```

**设计要点**：
- ✅ 使用 goroutine 并发扫描（可配置并发数）
- ✅ 断点续传：记录最后处理的文件路径
- ✅ 速度控制：每日上限、时间窗口限制
- ✅ 错误容错：单张失败不影响整体

---

### 3.3 AI 分析模块 (AIService)

**职责**：
- 调用 Qwen-VL API 分析照片
- 生成描述、文案、分类
- 计算评分
- 错误重试
- 成本统计

**分析流程**：
```
获取照片
  ↓
读取文件 + EXIF 信息
  ↓
构造 Qwen API 请求
  ├─ 照片 base64 编码
  ├─ prompt 模板
  └─ temperature 参数
  ↓
调用 Qwen API
  ↓
解析响应 JSON
  ├─ caption (80-200字)
  ├─ side_caption (8-30字)
  ├─ category (主分类)
  ├─ type (辅助标签)
  ├─ memory_score (0-100)
  ├─ beauty_score (0-100)
  └─ reason (评分理由)
  ↓
保存到数据库
  ↓
统计 API 调用次数和成本
```

**设计要点**：
- ✅ 异步处理：使用消息队列（Go channel）
- ✅ 速率控制：每日上限、令牌桶算法
- ✅ 错误重试：网络错误重试 3 次，指数退避
- ✅ 成本监控：实时统计调用次数和预估成本

### 3.4 展示服务模块 (DisplayService) ⭐

**职责**：
- 实现"往年今日"算法
- 照片去重（7天内不重复）
- 多次请求返回不同照片
- 记录展示历史

**"往年今日"算法**：
```go
func (s *DisplayService) GetTodayPhoto(deviceID string) (*DisplayPhoto, error) {
    today := time.Now()

    // 策略 1: ±3 天
    photos := s.findPhotosInRange(today, 3)
    if len(photos) > 0 {
        return s.selectPhoto(photos, deviceID, "on_this_day")
    }

    // 策略 2: ±7 天
    photos = s.findPhotosInRange(today, 7)
    if len(photos) > 0 {
        return s.selectPhoto(photos, deviceID, "on_this_week")
    }

    // 策略 3: 本月
    photos = s.findPhotosInMonth(today)
    if len(photos) > 0 {
        return s.selectPhoto(photos, deviceID, "on_this_month")
    }

    // 策略 4: 年度最佳
    photos = s.findTopPhotos(85.0, 10)
    return s.selectPhoto(photos, deviceID, "high_score")
}
```

**设计要点**：
- ✅ 降级策略：确保总能返回照片
- ✅ 去重机制：记录展示历史，避免重复
- ✅ 多次请求：同一天多次请求返回不同照片
- ✅ 设备隔离：不同设备独立去重
