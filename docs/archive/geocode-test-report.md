| 上海外滩 | 浦东新区（东岸绿道）| 浦东新区 | - | 东岸绿道 | ⚠️ 缺市级 |
| 纽约 | 纽约;紐約Manhattan（7th Avenue）| 纽约;紐約 | Manhattan | 7th Avenue | ✅ 较好 |

### 特点
- **速度**: 1-2.5s ⚠️ 慢（需遵守1秒限制）
- **国内精度**: ⭐⭐ 仅到区/县，无省市
- **海外覆盖**: ⭐⭐⭐⭐ 较好
- **地址层级**: city=区/县, suburb=街道, road=道路

### 限制说明
Nominatim 对中国地址的数据结构不同：
- 不提供省级/市级信息
- `city` 字段 = 区/县（如"朝阳区"）
- `suburb` 字段 = 街道（如"三里屯街道"）
- 无法生成"北京市朝阳区"格式

## 3. 离线数据库 (Offline)

### 特点
- **速度**: 1-5ms ✅ 极快
- **数据源**: GeoNames cities500
- **城市名**: 英文/拼音
- **地址层级**: 仅国家/省/城市
- **街道/商圈**: ❌ 不支持

### 示例输出
```
中国，北京市，Beijing
```

## 配置建议

### 国内用户（推荐）
```yaml
geocode:
  provider: "weibo"      # 主要：微博RGC
  fallback: "offline"    # 备用：离线数据库
```

### 海外用户
```yaml
geocode:
  provider: "nominatim"  # 主要：Nominatim
  fallback: "offline"    # 备用：离线数据库
```

### 混合方案
```yaml
geocode:
  provider: "weibo"      # 国内优先微博
  fallback: "nominatim"  # 海外回退Nominatim
```

## 已知问题

1. **微博 RGC 杭州西湖 POI 识别为"浙江"**
   - 可能是坐标落在较宽泛区域
   - 建议：街道信息"米市巷街区"已足够精确

2. **Nominatim 缺少中国省市信息**
   - 数据源限制，无法修复
   - 建议：国内场景优先使用微博

3. **海外坐标微博支持有限**
   - 纽约仅返回城市名
   - 建议：海外照片使用 Nominatim
