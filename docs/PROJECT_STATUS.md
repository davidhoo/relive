# Relive 项目状态报告

> 报告日期：2026-04-02
> 当前版本：v1.3.0（准备中）
> 状态：核心功能完备，包含人物系统、事件策展、照片旋转、Web / Backend / Analyzer / ESP32 固件

## 当前结论

- 后端服务已具备完整的 Web 管理、照片扫描、AI 分析、缩略图、地理编码、设备管理、事件聚类与策展展示能力
- 前端管理后台已覆盖仪表盘、照片、人物、AI 分析、缩略图、GPS、事件浏览、设备、展示策略、配置、系统管理等页面
- `relive-analyzer` 已切换为 API 模式，不再以 `export.db` 导入导出为当前默认工作流
- ESP32 墨水屏相框固件已完成，支持 AP 配网与定时睡眠调度
- 事件驱动型智能策展引擎已完成（6 通道提名 + 多样性选择 + 序列编排）
- 人物系统一期已完成：异步人脸检测队列、人物聚类/纠错、人物列表与详情、照片详情人物区、展示策略人物权重

## v1.2.1 以来的主要变更

### 人物系统（核心新功能）
- 新增 `faces` / `people` / `people_jobs` 数据结构与异步人物处理队列
- 新增 `/api/v1/people` 系列管理接口、`/api/v1/photos/:id/people` 聚合接口、`/api/v1/faces/:id/thumbnail` 鉴权裁切图接口
- 前端新增人物管理页（列表 + 后台任务）和人物详情页（改名、改类别、改头像、拆分、移动、合并）
- 照片详情页可直接查看该照片中的人物及人脸样本
- 展示策略开始读取 `photos.top_person_category`，`people_spotlight` 优先使用真实人物数据而不是仅靠标签猜测

### 事件系统（核心新功能）
- 基于时空连续性的照片事件聚类算法
- 多维事件策展引擎（6 通道：时光隧道 / 巅峰回忆 / 地理漂移 / 角落遗珠 / 人物专题 / 季节专题）
- 前端事件浏览页面（卡片列表 + 详情照片网格）
- 展示策略页接入策展配置 + 批次详情标注来源通道

### 照片旋转
- 照片方向手动覆盖（manual_rotation 字段）
- 缩略图和批次渲染正确应用旋转

### 展示改进
- 批次生成写入展示记录 + 预览会话级排除
- 修复跨日照片重复、切换日期预览不刷新等问题

### 稳定性
- SQLite 并发锁竞争优化
- SyncTags 嵌套事务死锁修复
- analyzer 损坏图片永久跳过、优雅退出

### 前端体验
- 照片管理页 Shift 范围多选 + 全选/反选
- 事件详情返回列表保留分页状态

### 代码质量
- 拆分 photo_service.go 和 display_service.go
